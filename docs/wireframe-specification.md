# Lab 05 - Wireframe Specification

Each required screen, what it must contain, and the requirement it satisfies. Wireframes are exported as screenshots of the working web prototype.

## 1. Homepage (wireframes/homepage.png) — FR-01, FR-02
Project title "Student Apartment Finder," target user and problem statement, a search bar, and navigation (Home, Apartments, Roommates, Saved, About, Sign in).

## 2. Input Form — Owner Add Listing (wireframes/input-form.png) — FR-03, FR-10, FR-11
Fields: apartment name, price, distance from campus, room type, facilities, availability, photos, owner contact. Required-field validation and a "Listing submitted" confirmation.

## 3. Record List — Apartments (wireframes/record-list.png) — FR-05, FR-06
Grid/list of apartment cards with search box and filters (budget, distance, room type, university). Each card shows price, distance, and status.

## 4. Detail View — Apartment Detail (wireframes/detail-view.png) — FR-07, FR-08
Full listing: photos/gallery, price, distance, room type, facilities, description, availability status badge, owner contact.

## 5. Dashboard — Admin Dashboard (wireframes/dashboard.png) — FR-12
Summary metrics: total listings, available rooms, pending approvals, roommate posts; simple charts.

## 6. Admin View — Admin Listings (wireframes/admin-view.png) — FR-09, FR-08
Table of listings with approve / reject / remove actions and status (Approved / Pending / Rejected).

## Consistency (FR-13) and Responsiveness (FR-14)
All screens share the same navigation, layout, buttons, and colours, and adapt to laptop and mobile widths.
